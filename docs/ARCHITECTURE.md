# NyayaAI Architecture

## Overview

NyayaAI follows **Clean Architecture** with a monorepo structure, separating concerns across frontend, backend, shared packages, and database layers.

## Domain Models

### Core Entities
- **User** вҖ” Multi-role (Citizen, Advocate, Admin) with RBAC
- **Case** вҖ” Legal case management with timeline and notes
- **Document** вҖ” AI-generated and user-uploaded legal documents
- **Conversation** вҖ” AI chat sessions with citation metadata
- **Judgment** вҖ” Supreme Court and High Court case law
- **Legislation** вҖ” Indian statutes and acts
- **ConstitutionArticle** вҖ” Constitutional provisions with vector embeddings
- **AdvocateProfile** вҖ” Verified advocate marketplace profiles
- **Appointment** вҖ” Consultation booking and scheduling

## AI Pipeline

1. **Query Processing** вҖ” Natural language understanding of legal queries
2. **RAG Retrieval** вҖ” Semantic search across legal knowledge base using pgvector
3. **LLM Generation** вҖ” OpenAI GPT-4 / Gemini with structured JSON output
4. **Citation Assembly** вҖ” Constitutional, statutory, and case law references
5. **Response Formatting** вҖ” Structured legal guidance with disclaimers

## Security

- JWT access + refresh token rotation
- bcrypt password hashing (12 rounds)
- Helmet security headers
- Rate limiting (100 req/15min)
- RBAC with role-based route guards
- Audit logging for admin actions
- Input validation with Zod schemas

## Scalability

- Stateless API servers (horizontal scaling)
- PostgreSQL read replicas for research queries
- Redis for session caching and rate limiting
- CDN for static assets (Vercel)
- S3 for document storage
- Vector database for semantic legal search
